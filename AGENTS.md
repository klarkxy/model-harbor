## 产品定位

- 产品名：ManageYourLLM。
- 类型：个人自用 LLM 网关与管理台。
- 目标：稳定接入、顺序 failover、可解释 trace、成本账本和可恢复备份。
- v1.0.0 完成定义：空库启动后，通过 Setup Wizard 完成 Provider -> Endpoint -> Model/Channel -> Client -> 网关测试 -> Usage/Trace -> Backup 的闭环。

明确不做：

- 组织平台、多租户、RBAC、OIDC、SSO。
- 充值、售卖、支付、分账、企业账单。
- PostgreSQL。
- 旧数据库自动迁移。
- Codex OAuth。
- 本机客户端配置自动写入。
- weighted / round-robin / cost-aware / quality-aware routing。

## 核心概念

```text
Provider Account 是认证/账号边界。
Endpoint 是协议/健康/能力/路由边界。
Model 是客户端可见模型边界。
Channel 是客户端可见用途边界。
Candidate 是 failover 边界。
Client 是接入凭证边界。
```

命名约定：

| 旧词               | 新词                        |
| ------------------ | --------------------------- |
| Upstream Key       | Provider Account / 上游账号 |
| Public Model       | Model / 模型                |
| Model Group        | Channel / 频道              |
| App                | Client / 客户端             |
| Consumer Key       | Client key                  |
| Debug Content Logs | Trace 临时内容记录          |

## v1 UI 结构

左侧导航按组：

```text

  Overview

  Providers

观测排障
  Usage
  Traces

成本管理
  Costs

运维

```

Settings 只放系统级参数。Backups、Costs、Clients、Models 都是独立业务页面。Debug content 合入 Traces，不做独立导航。

## 路由原则

- v1 只做顺序 failover。
- 用户排序决定尝试顺序。
- Costs 不参与路由。
- Channel 是有序 Model 集合，不做复杂策略。
- Candidate 级 cooldown / circuit breaker / endpoint health 参与过滤。
- `bad_request`、`auth`、`permission`、`model_not_found` 应进入 Trace 和配置风险提示，不要轻易冷却整个 Provider Account。
- 公开网关 base path 固定为 `/v1`，不再暴露 `gatewayBasePath` 配置。

## 技术栈

- 数据库：SQLite + Drizzle/libsql。
- 前端：Vue 3 + Vite + Naive UI + Pinia。
- Monorepo：pnpm workspace。

当前代码仍有 application 直接依赖 infrastructure row/repository 的历史债。改 routing/gateway 主链路时，优先引入运行时 snapshot，如 `ProviderAccountSnapshot`、`EndpointSnapshot`、`CandidateSnapshot`、`RoutingSettingsSnapshot`。

## 开发约定

- 文档、UI、注释以中文优先，保留 i18n 结构。
- Provider preset 是模板，用户创建 Provider Account 后可以自由改 endpoint。
- Client 首版一个 active key，不做模型权限，不做 client type。
- Model / Channel 是客户端可请求名称，Candidate 是内部详情。
- 数据写入统一经过 repository / unit-of-work；server routes 不直接写复杂数据库逻辑。
- Secret 加密落库；Client key raw value 只在创建/轮换时显示一次。
- 内容日志默认关闭，只能作为 Trace 的临时内容记录能力，必须有过期时间、最大条数、脱敏和截断。
- 修改本文件描述的文件、结构或设计后，必须同步更新本文件。
