# 用量与监控

ModelHarbor 提供多种观测手段，帮助管理员了解系统运行状态、用量分布和潜在问题。

---

## 管理后台 Overview

登录后首页展示核心指标：

- 今日请求总数
- 今日输入 / 输出 / 总 token 数
- 活跃上游密钥数
- 冻结上游密钥数
- 最近失败请求
- 会话粘性命中率
- 需要关注的上游密钥

---

## Usage 页面

**Usage** 页面提供按时间窗口筛选的用量统计和明细：

### 时间窗口

- `today`
- `24h`
- `7d`

### 汇总维度

| 维度 | 用途 |
| --- | --- |
| 按应用 | 查看每个 App 的请求量和 token 消耗。 |
| 按 Consumer Key | 追踪具体 key 的使用情况。 |
| 按上游密钥 | 查看每个上游 key 的调用量和失败率。 |
| 按目标 | 查看每个公共模型或模型组的使用情况。 |

### 最近请求

展示最近请求列表，包含：

- 目标模型名
- 上游密钥
- 状态（成功 / 失败）和错误码
- 延迟
- trace ID

> 默认不展示 prompt 和 completion 内容。如需查看完整内容，请开启内容日志并在对应模块查询。

---

## 链路追踪（Request Trace）

每次网关请求会生成唯一的 `traceId`，并在响应头 `X-Request-Trace-Id` 中返回。

在 **Usage → 查看链路** 中，可以按 trace ID 查看完整链路时间线：

- 请求开始
- 认证结果
- 目标解析
- 候选展开与过滤
- 粘性检查
- 每次候选尝试
- 成功 / 失败 / 冷却
- 流式开始/结束

链路日志默认保留 30 天，用于排查“为什么这次请求走了这条路”“为什么某个候选被过滤”。

---

## 端点健康

在 **Upstream Keys** 列表或详情页，可以查看每个 endpoint 的探测状态：

- 延迟
- 是否降级
- 最近检查时间
- 错误码和错误信息

如果某个 endpoint 持续降级，系统会优先把流量切到其他健康 endpoint。

---

## 熔断器监控

**Circuit Breakers** 页面展示所有处于 `open` 或 `half_open` 状态的熔断器：

- 上游密钥名
- 真实模型名
- 当前状态
- 失败次数
- 冷却结束时间
- 最后错误类型

可以手动重置误判或长期无法自动恢复的熔断器。

---

## 内容日志

默认情况下，ModelHarbor **不记录**完整 prompt 和 completion。如果业务需要审计或调试，可以在 **Settings → Content Logging** 中开启：

- **Enabled**：开关。
- **Retention days**：保留天数。
- **Max payload bytes**：单行最大字节数，超出部分截断。

开启后，系统会在写入前进行脱敏：移除 `mh_`、`sk-`、`Bearer` 等敏感 token。

> 内容日志可能包含用户数据，请确保符合所在组织的隐私合规要求。

## 日志与审计

### 运行日志

默认记录：

- 管理员登录成功/失败
- 管理员资源变更（上游密钥、模型、Consumer Key 等）
- 配额冻结/解冻、冷却事件

### 审计事件

关键管理操作会写入审计日志：

- 管理员登录/登出
- 上游密钥创建、更新、冻结、解冻、轮换
- 公共模型创建、更新、禁用
- 模型组创建、更新、禁用
- Consumer Key 创建、撤销、轮换
- Consumer Key 授权变更

---

## 告警建议

建议对以下指标配置告警：

- `/readyz` 返回非 200
- 上游密钥失败率持续升高
- 熔断器频繁打开
- 某个 endpoint 持续降级
- 上游密钥配额即将耗尽
- 错误率突增
- 管理员登录失败次数突增

---

## 排查思路

如果用量或监控数据异常：

1. 先看 **Overview** 的总览卡片，定位是失败率高还是某个上游 key 异常。
2. 按 **Upstream Key** 维度查看失败分布。
3. 找到具体失败的 trace ID，查看链路详情。
4. 结合 **Circuit Breakers** 和 **Endpoint Health** 判断是上游问题还是配置问题。
